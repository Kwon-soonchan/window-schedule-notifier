# 윈도우 일정 알리미 (Window Schedule Notifier)

## 📅 1. 프로젝트 소개

이 프로젝트는 우테코 8기 프리코스 이후 진행된 오픈 미션 프로젝트입니다.
대학 생활과 과제를 병행하며 수동으로 일정을 관리하던 불편함을 해결하기 위해 시작했습니다.

지정한 날짜와 시간이 되면 윈도우 네이티브 알림을 띄워주는 데스크탑 일정 관리 프로그램입니다.

## ✨ 2. 핵심 기능

* **MVC 아키텍처:** 데이터(`Model`), 화면(`View`), 로직(`Controller`)을 분리하여 유지보수성을 높였습니다.
* **백그라운드 알림:** `controller.Main`이 1분마다 일정을 감시하여, 설정한 시간과 일치하면 윈도우 알림을 띄웁니다.
* **일정 관리 UI:** `view.ScheduleManagerUI`를 통해 일정을 관리합니다.
    * **일정 추가:** 날짜와 시간을 포함한 일정 등록 (예: `2025-11-24 14:30,과제 제출,오늘까지`)
    * **일정 삭제:** 특정 시간의 일정을 찾아 삭제
    * **자동 정렬:** 입력 순서와 관계없이 날짜/시간순으로 자동 정렬되어 표시
* **공용 데이터 관리:** `user.home` 경로를 사용하여 UI와 백그라운드 프로세스가 동일한 데이터 파일을 공유합니다.

## ⚙️ 3. 사용 기술

* **Language:** Java 21
* **Architecture:** MVC Pattern
* **UI:** Java `Swing` (`JFrame`, `JTextArea` 등)
* **Core Logic:**
    * `java.time.LocalDateTime`: 날짜 및 시간 데이터 처리
    * `java.awt.SystemTray`: 윈도우 시스템 트레이 및 알림 제어
    * `java.util.Timer`: 스케줄링 (1분 주기 체크)
    * `java.util.stream`: 데이터 필터링 및 변환 (삭제 로직 등)

## ⚠️ 4. 개발 과정 및 어려웠던 점 (Troubleshooting)

이 프로젝트를 진행하며 단순 기능 구현을 넘어, 아키텍처와 시스템 환경에 대한 깊은 고민을 했습니다.

1.  **치명적인 파일 경로 버그와 해결:**
    * **문제:** UI에서 일정을 추가해도 알림이 울리지 않는 현상이 발생했습니다. 확인 결과, UI는 프로젝트 루트의 파일을 수정하고, 백그라운드 앱은 빌드 폴더(`out/production`)의 파일을 읽고 있어 **서로 다른 파일**을 바라보고 있었습니다.
    * **해결:** `System.getProperty("user.home")`을 사용하여 두 프로그램이 OS 사용자 홈 디렉토리의 **단일 공용 파일**을 참조하도록 경로를 통일하여 해결했습니다.

2.  **MVC 패턴 리팩토링:**
    * 초기에는 `Main`과 `UI` 클래스에 파일 입출력 로직이 혼재되어 있었습니다.
    * 유지보수를 위해 **Model**(`Schedule`, `ScheduleRepository`), **View**(`UI`), **Controller**(`Main`, `Listener`)로 역할을 명확히 분리했습니다. 덕분에 날짜 기능을 추가할 때 `Model`만 수정하면 되어 확장성이 크게 좋아졌습니다.

3.  **`LocalDateTime` 도입:**
    * 처음엔 `LocalTime`(시간)만 다뤘으나, 현실적인 일정 관리를 위해 `LocalDateTime`(날짜+시간)으로 업그레이드했습니다.
    * `DateTimeFormatter`를 활용해 파싱 로직을 강화하고, 날짜가 포함된 문자열 정렬(`Collections.sort`)이 자동으로 된다는 점을 활용해 로직을 단순화했습니다.

4.  **아이콘 리소스 로딩 문제:**
    * `src` 폴더 내의 이미지를 `ImageIcon`으로 로드할 때 경로 문제로 검은 네모가 뜨는 현상을 겪었습니다. `Class.getResource()`를 통해 클래스패스 기준으로 리소스를 안전하게 로드하도록 수정했습니다.

## 🚀 5. 실행 방법

1.  이 프로젝트를 Clone 받습니다.
2.  **알림 기능 실행:** `controller.Main`을 실행합니다. (백그라운드에서 상시 실행되며 트레이 아이콘이 생성됩니다.)
3.  **일정 관리:** `view.ScheduleManagerUI`를 실행합니다.
4.  **테스트:**
    * UI에서 미래 시간(예: 1~2분 뒤)으로 일정을 추가합니다. (예: `2025-11-24 15:00,테스트,알림 확인`)
    * 지정한 시간이 되면 윈도우 알림이 뜨는지 확인합니다.
